## 1. 工作概述

### 1.1 项目背景

理论文档提出"**量子全局视野 + 经典局部贪心**"的双层算法框架，核心研究问题：

> 连续时间量子游走（CTQW）产生的节点概率分布，能否比传统局部指标
> 更好地反映图的全局拓扑结构，并提升贪心算法在图组合优化问题上的求解质量？

围绕此问题，理论文档 §3 提出三条可证伪的核心假设：

| 假设 | 内容 | 对应实验 |
|---|---|---|
| **H1** | CTQW 概率能够反映图的全局结构（在团/密集子图区域形成概率集中） | 实验一 |
| **H2** | 量子概率评分能够改善贪心选择质量 | 实验二 |
| **H3** | 混合评分比单一评分更稳定 | 实验三、实验四 |

### 1.2 本次工作内容

1. **工程实现**：完成 CTQW 矩阵指数计算 `|ψ(t)⟩ = exp(-iHt)|ψ₀⟩` 的真实实现，
   替换原占位代码（基于 `scipy.linalg.expm`）；
2. **数值正确性验证**：编写冒烟测试脚本，确认 Hermitian、幺正性、概率守恒等 6 项；
3. **修复算法缺陷**：发现并修复 `DenseCandidateSet` 缺少停止条件的 bug，
   补全 DS 任务的 `|S| = k` 停止逻辑；
4. **运行四组实验**：按理论 §13 的实验设计依次完成实验一至实验四；
5. **批量统计分析**：累计运行约 12,500 次算法实验，覆盖
   人工 planted 数据集 n∈{30, 50, 100, 150, 200}、5 个 α 取值、6 个 λ 取值、5 个 t 取值；
6. **显著性检验**：引入 Wilcoxon signed-rank paired test，按 (sample_id, run_id) 配对，
   严格判定算法间差异的统计显著性。

## 2. 实验环境与实现说明

### 2.1 软件环境

- Python 3.13
- NumPy 2.4 / SciPy 1.17 / NetworkX 3.6 / Matplotlib 3.11 / Pandas 3.0

### 2.2 CTQW 演化的实现

按理论 §5 §6 定义，单轮 CTQW 概率计算流程：

```python
H = construct_hamiltonian(adjacency, S, lam)   # H = A + λ·Σ|u⟩⟨u|
psi0 = build_initial_state(n, S, adjacency, init_method)
U = scipy.linalg.expm(-1j * H * t)             # 演化算子
psi_t = U @ psi0                               # 状态演化
probs = np.abs(psi_t) ** 2                     # 概率 P_v = |ψ_v|²
```

### 2.3 数值正确性验证

冒烟测试在 `mc_n30_p01_k5_000` 实例上检查 6 项：

| 检查项 | 结果 |
|---|---|
| H = H† （Hermitian） | max\|H − H†\| = 0.00 |
| U†U = I （幺正性） | max\|U†U − I\| ≈ 1e-15（机器精度） |
| Σ P_v = 1 （概率守恒） | 误差 < 1e-15 |
| P_v ≥ 0 （概率非负） | min P_v = 7.22e-4 |
| Ratio > 1 （假设一） | 最高 Ratio = 9.43 |
| QuantumGuidedGreedy 端到端 | 精确找到植入团 {12,13,24,27,28} |

**全部 6 项通过，CTQW 实现数值上正确**。

### 4.1 实验目的

| α | 团大小 μ±σ | Δ vs 基线 |
|---|---|---|

## 5. 实验三：消融实验（验证 H3 的"贡献度拆解"维度）

|---|---|---|---|---|

## 6. 针对实验二的深入理论分析

实验二的结果出乎我们的预期：QuantumGuidedGreedy 在所有规模、所有参数下都
输给了经典的 ClassicalClique。这一节用更直观的方式解释为什么会出现这种现象。

### 6.1 在解释机理之前：贪心算法每轮在做什么？

最大团贪心算法是一个**迭代扩团**的过程。算法维护一个"当前已选节点集合 S"，
每一轮做三件事：

1. **找候选**：列出"能继续加入团的节点"——即与 S 中所有节点都有边相连的节点；
2. **打分**：用某个评分函数给每个候选节点打分；
3. **选最高**：把得分最高的节点加入 S。

直到没有节点能继续加入，算法停止，返回最终的 S 作为找到的团。

**两种算法的区别只在第 2 步打分函数上**：

- **ClassicalClique** 的 R 评分：候选节点 v 在"剩余候选诱导子图"里的度数。
  直观含义：选了 v 之后，还有多少节点能继续往团里加？
- **QuantumGuidedGreedy** 的 Q 评分（α=1 时）：CTQW 演化后，候选节点 v 的概率 P_v(t)。
  直观含义：量子游走从 S 出发"扩散"后，概率集中在 v 的程度。

也就是说，**两种评分都是在问"这个候选节点跟当前 S 的关系有多紧密"**，
只是问的方式不同——R 是"硬性数边"，Q 是"软性算量子概率"。

### 6.2 之前给出的三种解释——以及它们为什么不对

在初步分析中，我们曾尝试用这三种解释，但都被实验数据否定。

#### 解释 1：「ClassicalClique 基线太强，QuantumGreedy 拼不过」

这个解释**听起来合理但不对**。如果只是"基线强"，那么换更难的图
（medium n=100~200）应当让差距缩小——因为更难的图上，"贪心+局部信号"
应该会失灵，把胜负差距拉平。

但**实验显示差距反而扩大**：

| 规模 | α=1.0 与基线的差距 |
|---|---|
| small (n≤50) | −3.06 |
| medium (n=60~200) | **−4.93**（扩大） |

如果是"基线太强"，medium 上应该缩小才对。所以这个解释不是根本原因。

#### 解释 2：「init=max_degree 没让 CTQW 发挥优势，改成 init=uniform 就好了」

这个解释**被代码实际行为否定**。打开 `src/initial_state.py:33` 可以看到：

```python
if S:
    # 基于种子集合的均匀初态
    amp = 1.0 / np.sqrt(len(S))
    for u in S:
        psi[u] = amp
    return psi
```

**只要 S 非空，初始态就是种子集合上的均匀叠加，与 init_method 参数无关**。

也就是说，init_method 只在第一轮（S=∅）起作用，对后续 4-5 轮的迭代完全没影响。
参数扫描确实显示：uniform 和 max_degree 给出了几乎相同的性能曲线。

#### 解释 3：「α=0.5 太民主，应该调小让 Q 影响小一点」

这个解释**被参数扫描的单调性否定**。我们扫了 α ∈ {0, 0.25, 0.5, 0.75, 1.0}：

| α | 团大小 | 相对基线 |
|---|---|---|
| 0.00 | 7.20 | +0.00（退化为经典）|
| 0.25 | 7.14 | −0.06 |
| 0.50 | 6.71 | −0.49 |
| 0.75 | 5.08 | −2.12 |
| 1.00 | 4.65 | −2.55 |

可以看到 **α 越大、性能越差**，是单调下降的。**不存在"中间最优区间"**——
任何 α > 0 都让性能更差。这意味着加入 Q 评分本身就是负贡献，不是"权重没调对"的问题。

---

### 6.3 真正的机理：Q 评分跟 R 评分在做同一件事，但 Q 做得更差

#### 关键洞察：理论 §6 的初始态选择决定了 Q 评分的"方向"

理论文档 §6 定义贪心迭代中的初始态为：

$$|\psi_0\rangle = \frac{1}{\sqrt{|S|}} \sum_{u \in S} |u\rangle$$

也就是"种子集合上的均匀叠加"。从物理上看，这是一个"**从已选节点 S 出发**"
的量子态。

CTQW 演化算子 $U(t) = e^{-iHt}$ 让这个量子态向外扩散。演化时间 t 后，
高概率会集中在"**与 S 强相连、可达性高**"的节点上——
即"从 S 走一步能到、走两步也能到的节点"。

#### 但这正是 R 评分要找的节点

R 评分 `R(v, S) = deg_{C(S)}(v)` 计算的是：
"候选节点 v 在剩余候选诱导子图里的度数"。

候选集合 C(S) 本身已经过滤为"与 S 中所有节点都相连的节点"。
所以 v 的"候选诱导子图度数"实际上回答：
"如果我选了 v，下一轮还有多少节点能继续加入团？"

**这个问题的答案与"哪些节点跟 S 高度连接"高度相关**——因为如果一个候选节点
v 同时跟许多其他候选连接，说明它处于"高连接度区域"，正是 CTQW 概率
也会高的地方。

#### 一个简单的例子

假设当前 S = {a, b}，候选 C(S) = {c, d, e}，其中：
- c 与 d、e 都有边（在候选诱导子图里度数 = 2）
- d 只与 c 有边（度数 = 1）
- e 只与 c 有边（度数 = 1）

R 评分判断：选 c，因为它后续扩展空间最大。

CTQW 评分判断：从 |ψ₀⟩ = (|a⟩+|b⟩)/√2 出发演化。概率会扩散到 c, d, e。
由于 c 是 d 和 e 的"必经枢纽"，概率经多步路径加和后**也会在 c 处更高**。

**两种评分的判断是一致的——都选 c**。

那为什么 Q 评分会拖累性能？因为：

1. **当候选都不是明显赢家时，Q 评分受相位干涉影响，可能出错**。
   CTQW 概率不是简单的"路径数加和"，而是带相位的复数幅相加。
   相位可能"相消"——两条路径的贡献抵消，导致明明高连接的节点反而概率不高。

2. **R 评分是确定的整数计数**，没有上述抵消问题。在小图上 R 几乎不会出错。

3. **混合评分 α·Q + (1−α)·R 让 Q 的"偶尔出错"污染了 R 的"几乎不出错"**。

#### 数据佐证：实验三验证了"R 是关键，Q 单独无用"

实验三的消融数据正好支撑上述机理：

| 组 | 配置 | 团大小 | 解读 |
|---|---|---|---|
| A | 纯 R 评分 | 7.20 | R 单独工作就接近最优 |
| B | 纯 Q 评分，λ=0 | 4.71 | Q 单独工作只能做到一半水平 |
| C | 纯 Q 评分，λ>0 | 4.65 | 加 λ 没救 |
| D | Q + R 混合 | 6.71 | 混合不如纯 R |

**R 评分在 small 上已经做到 7.20（接近 5-团之上的扩展极限），Q 没有
任何"R 没看到但 Q 能看到"的额外信息**——它能给的提示 R 都已经给了，
而且更精确。

### 6.4 为什么实验一显示 Ratio = 7.08，但贪心里 Q 还是没用？

这是最关键、也最反直觉的一点。

#### 实验一用的是 S=∅ 下的"全图均匀叠加"

实验一计算 Ratio 时，使用的初始态是：

$$|\psi_0\rangle = \frac{1}{\sqrt{n}} \sum_{v \in V} |v\rangle$$

即**所有 n 个节点同时作为出发点**，演化后看哪些节点概率集中。
这种全图叠加让"图中所有路径同时干涉"，最终在拓扑密集的区域
（如团内部）相干增强。

**这是"全局拓扑识别"任务**——CTQW 看整张图，找出哪里最"紧密"。
实验一证明在小图上它能完成这个任务，Ratio = 7.08 就是证据。

#### 贪心算法实际用的是"种子集合均匀叠加"

但贪心算法进入第二轮之后，S 已经有节点了，初始态变成：

$$|\psi_0\rangle = \frac{1}{\sqrt{|S|}} \sum_{u \in S} |u\rangle$$

**这是完全不同的物理量**：
- 全图均匀叠加 → 全局识别（实验一验证有效）
- 种子集合均匀叠加 → 局部种子扩散（实验二实际场景）

打个比方：
- 实验一像是"在地图上同时点亮所有城市，看哪些区域光最亮"——
  用来识别人口最密集的城市群（团）；
- 实验二像是"只在 A、B 两个城市点亮，看光会怎么传到周围"——
  用来选下一个跟 A、B 关系紧密的城市；

**这两件事用的是同一种灯（CTQW），但回答的是完全不同的问题**。

实验一证明 CTQW 能完成"全局识别"任务，但贪心算法用的是 CTQW 的
"局部扩散"功能——而这个功能正是 R 评分已经做得很好的事。

### 6.5 综合结论

把上述四点合起来：

> **CTQW 自身没有错（实验一验证有效，数值正确性 6/6 通过），
> 错的是"按理论 §6 定义的方式把 CTQW 嵌入贪心算法"。**
>
> 当贪心的初始态变成种子集合均匀叠加后，CTQW 提供的信号
> （"哪些节点跟当前 S 强相连"）**与 R 评分提供的信号方向完全重合**，
> 但因为相位干涉效应，Q 评分的精度低于 R 评分。
> 加之候选集合 C(S) 本身已经做了"可行性筛选"，留给 Q 的辨别空间被
> R 完全覆盖——结果就是 Q 不仅没贡献，还会污染 R 的判断。

这是本次实验在理论文档基础上揭示的关键发现：
**H1 的"全局识别能力"与 H2 的"贪心扩展能力"是两个不同的任务，
H1 成立不蕴含 H2 成立**。

## 7. 总结：三个假设的最终验证情况

| 假设 | 状态 | 关键证据 |
|---|---|---|
| **H1** CTQW 反映全局结构 | **✓ 成立** | 实验一：Ratio = 7.08（小图） |
| **H2** 量子评分改善贪心质量 | **✗ 不成立** | 实验二：α 单调递减；medium 上差距扩大；细分指标全输 |
| **H3** 混合评分比单一评分稳定 | **⚠ 部分成立** | 实验三：D > B/C（+43%）✓；但 D < A（−6.8%）✗ |

**整体性结论**：CTQW 在小规模图上确实能识别团区域（H1 成立），
但当 CTQW 被嵌入贪心扩展算法、初始态从全图均匀变为种子集合均匀后，
它与经典 candidate-subgraph degree 评分形成了"同方向但更低精度"的关系，
反而削弱了贪心算法的判断质量（H2 不成立）。
混合评分能缓解纯量子方案的崩溃（H3 在 B/C/D 比较中成立），
但当对手是强经典基线时，混合方案仍非最优。

## 8. 后续可探索的优化方向

本节列出 4 个**具体、可独立实现、互不依赖**的优化尝试方向。
每个方向都明确指出：要改哪个文件的哪个函数、改成什么样、如何验证。

> **重要说明**：本节列出的是"探索性优化"，
> **第 3-7 章已有的实验结果和结论是完整、可交付的**。
> 即使本节的优化全部尝试都失败，也不影响本次实验报告的核心结论。
> 这里列出的优化都是**在现有结果基础上的"加分项"**，用来探索
> "如果调整 Q 评分的提取方式，能否让 H2 在某些场景下成立"。

---

### 8.1 优化方向 A：去均匀基线的 Q 评分

**改动位置**：`src/scoring.py` 的 `QuantumScorer.score_all` 方法

**当前实现**：

```python
probs = np.abs(psi_t) ** 2
return {v: float(probs[v]) for v in candidates}
```

**改动建议**（增加一行）：

```python
probs = np.abs(psi_t) ** 2
n = instance.num_nodes
probs = probs - 1.0 / n   # 去均匀基线，让"无信息节点"为零
return {v: float(probs[v]) for v in candidates}
```

**改动原理**：

当前 Q 评分用原始的 P_v(t)，最小值接近 0、最大值约 0.1，
归一化后大部分节点都被压到 0 附近，区分度低。
改成 `P_v(t) - 1/n` 之后，
**比均匀分布更高的节点为正、更低的节点为负**，
对比度更明显，归一化后区分更清晰。

**验证方法**：

```bash
python experiments/exp2_algorithm_comparison.py --task maximum_clique --small
```

看 α=0.5 时 QuantumGuidedGreedy 的均值。原来是 6.71，
看新实现能否拉近到 7.20（基线）。

**即使失败的情况**：

如果调完反而更差，说明"去基线"对评分没帮助，那就把改动恢复即可。
**不会影响第 3-7 章的实验结论**——那些结论是基于"按理论 §8.2 原始定义"得出的。

---

### 8.2 优化方向 B：用 Q 作为"打破平局"的二级排序

**改动位置**：`src/algorithms/quantum_greedy.py` 的 `solve` 方法

**当前实现**：

```python
combined = {
    v: self.alpha * q_norm.get(v, 0.0)
    + (1 - self.alpha) * r_norm.get(v, 0.0)
    for v in candidates
}
best_v = max(candidates, key=lambda v: combined[v])
```

**改动建议**：

```python
# 先按 R 评分排序，R 相同时用 Q 打破平局
best_v = max(candidates, key=lambda v: (r_scores[v], q_scores[v]))
```

**改动原理**：

实验三显示 R 评分是性能主力，Q 评分只是污染了 R 的判断。
那不如让 R 完全主导决策——只在"两个候选 R 分数完全一样时"才用 Q 决胜负。

这种"二级排序"方式让 Q 评分**只在 R 无法区分的情况下起作用**，
完全不会破坏 R 的好决策。

**验证方法**：

```bash
python experiments/exp2_algorithm_comparison.py --task maximum_clique --small
```

预期效果：QuantumGuidedGreedy 的均值应该 ≥ ClassicalClique 的 7.20。
如果出现 7.20+ε，说明 Q 在"打破平局"时确实提供了正贡献。

**即使失败的情况**：

如果均值还是 7.20（即 Q 在打破平局时也没用），那就说明
R 评分本身的平局情况已经很少，没给 Q 留下机会。
这本身就是一个有意义的发现，可以补充进报告。

---

### 8.3 优化方向 C：多时间平均的 Q 评分

**改动位置**：`src/scoring.py` 的 `QuantumScorer.score_all` 方法

**当前实现**：

```python
U = expm(-1j * H * self.t)
psi_t = U @ psi0
probs = np.abs(psi_t) ** 2
```

**改动建议**：

```python
# 用多个 t 值的概率求平均，减少单 t 的相位干涉抖动
t_values = [0.5, 1.0, 2.0, 5.0]
probs_sum = np.zeros(n)
for t in t_values:
    U = expm(-1j * H * t)
    psi_t = U @ psi0
    probs_sum += np.abs(psi_t) ** 2
probs = probs_sum / len(t_values)
```

**改动原理**：

实验四显示 t=2 时算法可能失败（找到 3-团而非 5-团），
原因是单个 t 下相位抵消可能让"应该高概率"的节点反而概率低。
多个 t 平均可以**抹掉这种抖动**，得到更稳定的"长期可达性"信号。

**验证方法**：

```bash
python experiments/exp4_sensitivity.py --task maximum_clique
```

看新实现的 α-λ 热力图上"失败区域"（α=0.75-1.0、λ 较小处的暗色块）
是否变少。

**即使失败的情况**：

多时间平均的代价是计算量变 4 倍。如果效果不明显，恢复成单 t 即可。
不影响主报告结论。

---

### 8.4 优化方向 D：解耦初始态，每轮都用全图均匀叠加

**改动位置**：`src/algorithms/quantum_greedy.py` 的 `_compute_ctqw_probs` 方法

**当前实现**：

```python
psi0 = build_initial_state(n, S, adjacency, self.init_method)
```

`build_initial_state` 在 S 非空时返回种子集合均匀叠加。

**改动建议**：直接构造全图均匀叠加，绕开 `build_initial_state`：

```python
# 改成每轮都用全图均匀叠加（实验一验证有效的初态）
psi0 = np.ones(n, dtype=np.complex128) / np.sqrt(n)
```

**改动原理**：

实验一证明全图均匀叠加下 CTQW 能识别团区域（Ratio = 7.08）。
但贪心迭代用的是种子集合均匀叠加（理论 §6 定义），
让 CTQW 退化成"局部种子扩散"。

如果改成每轮都用全图均匀叠加，则 CTQW 始终回答
"全图中哪些区域最像团？"这个问题。然后通过 λ 扰动项
`H = A + λ·Σ|u⟩⟨u|` 把 S 的影响传递给演化。

**验证方法**：

```bash
python experiments/exp2_algorithm_comparison.py --task maximum_clique --small
```

特别注意 α=1.0（纯量子）的表现。原来是 4.65，
如果新实现能拉到 6.0 以上，说明"换初始态"确实改善了 Q 评分的质量。

**即使失败的情况**：

如果性能没有提升甚至下降，说明"全图均匀叠加"在 S 已经有内容时
确实不太合理（CTQW 又"忘了"S 是谁了）。
这本身证明了理论 §6 设计种子集合均匀叠加的必要性。

### 8.5 这几个优化的关系

| 优化 | 难度 | 改动量 | 效果方向 |
|---|---|---|---|
| A 去均匀基线 | 简单 | 1 行 | 提升 Q 评分对比度 |
| B 二级排序 | 简单 | 1 行 | 让 R 主导，Q 只补漏 |

| D 改初始态 | 中等 | 1 行 + 测试 | 把 H1 的全局信号搬到贪心里 |

**建议执行顺序**：B → A → C → D（按"风险从低到高"）。
每完成一个就跑一次 `experiments/exp2_algorithm_comparison.py --small`
看效果。

**所有优化都不影响主报告**：第 3-7 章的所有数据、图表、结论都是基于
"按理论 §8 原始定义实现"的代码得出的。本节优化是**额外探索**，
即使全部失败也只是说明"理论 §8 的定义在我们的实验场景下已经是局部最优"——
这本身就是一个有意义的发现。
